Enabling discovery

Assigning identifiers to real world things and information resources

Mostly, people looking for information are interested in real world things, not in information records.

Real world things vs features: Lighthouse example goes here (insert whiteboard picture here); real world thing, feature is information record about the thing, but often in current practice has properties that are about the real thing and properties about the information record. The features are in a dataset, the dataset has an id and the features have a local id. So currently you cannot link to them; the features need to have http URI identifiers instead.

Examples: CSIRO spatial identifier reference framework; http://almere.pilod.nl/bgtld/v2; ...

tell people what to do if they've got feature data and publish that in a linked data way. First see if there's an authoritative URI already that you can link to and if so do that; else how to mint the identifier yourself based on the features in your dataset and how the identifier is for the real world thing the feature describes. Local names are not typically URIs. We know we want to use URIs as identifiers. Using sameAs to establish that two resources are about the same thing. These sameAs statements could be collected into one set and this set could have provenance so that you can see who created the sameAs statements and decide if you'd trust them.

datasets need to be identified

features (things) need to be identified

representations (geometry, topology) should be identified and must be if they are managed elsewhere.

Identifiers should be URLs (http URIs) and must be globally unique.

in order for identifiers to be useful, people should be comfortable creating them themselves (bottom up, like hashtags) and this BP will give some pointers about how to create good URIs (should be dereferenceable/resolvable). There is no top down authority that you have to go to in order to create identifiers for spatial objects. So just make them up yourself if you need them and they don't exist.
